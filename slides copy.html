03 Pipeline

Módulo 03 — Pipeline ETL + API

Separar OLTP de OLAP. Nada analítico toca Neon directamente.

Arquitectura to-be
Django App (/apply, /status, /api) │ reads fraud_results ▼ ┌─────────────┐ │ Neon (OLTP) │◄── pipeline escribe │ prod DB │ resultados + status └──────┬──────┘ │ ETL extrae ▼ ┌─────────────┐ │ DuckDB │◄── reglas corren AQUÍ │ (OLAP) │ └──────┬──────┘ │ CLI (Typer/Rich) Streamlit dashboard
Lo que construyen
  • ETL: Neon → DuckDB (incremental, sin reprocesar)
  • Reglas corren contra OLAP, no producción
  • Resultados escritos de vuelta a Neon
  • Status page muestra breakdown por regla
  • JSON API en /api/status/<id>/
  • CLI con Typer + Rich para ops
? La app dice "pending" para siempre. Las reglas corren contra prod.
✓ DuckDB corre el análisis sin tocar prod. El status page muestra la decisión real.
